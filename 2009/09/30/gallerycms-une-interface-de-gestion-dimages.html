GalleryCMS, une interface de gestion d'images

30/09/2009

GalleryCMS est-il vraiment un CMS ? Non, pas vraiment, il n'offre aucune fonctionnalité d'affichage d'images, pas de gestion utilisateurs,... Pourtant, GalleryCMS sert bien à la gestion de contenu ; il peut recevoir l'appelation de CMS.

GalleryCMS sert donc à classer des images par thématique ou catégorie ; une possibilité qu'offre n'importe quelle interface de gestion de photographies. Ce que propose GalleryCMS, c'est un fichier XML en sortie afin d'être disponible pour l'affichage dans d'autres applications, notamment du flash.

L'argument des codeurs de ce CMS est que, la plupart du temps, une interface en flash lit un fichier XML qui comporte les informations sur les images : adresse du fichier et légende. Cependant, il est difficile de maintenir cette base d'images et de proposer à un client de la modifier.

Typically, a Flash-based image gallery is supported by images that are referenced in a static XML file. The problem is that you need to have your customers trained properly to edit the XML file to manage their content. This is risky because the structure of XML is sensitive, which is likely to result in a structure error at some point when trying to manage several categories and dozens of images.
