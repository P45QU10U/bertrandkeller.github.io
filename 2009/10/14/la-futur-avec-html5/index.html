Le futur avec HTML5 par Bruce Lawson

14/10/2009

Le HTML4 n'a pas remplacé le HTML1 (celui qu'on utilise pour rédiger un billet de blog, par exemple), le HTML5 (HTML5 et l’avenir du web) ne remplacera pas le HTML4. Mais alors à quoi sert le HTML5 ? Bruce Lawson d'Opera (Opera, un navigateur pour personnes formidables) éclaire vos lanternes.

The Future of HTML5 by Bruce Lawson from Opera from Carsonified on Vimeo.

La spécification HTML5 était, à l'origine, dénommée "Applications Web 1.0". Ce document s'est concentré sur les nouvelles balises de marquage , petit à petit s'est inséré d'autres sujets tels que :

  1. Les images et les graphiques avec canvas
  2. La suppression de la validation de formulaires avec webforms 2.0
  3. Le stockage automatique en local de ses données
  4. La Géolocalisation
  5. La création de barre d'outils et de menus

Pour avoir un aperçu des possibilités de HTML5 :

  • Eye-candy canvas
  • canvas first-person shooter
  • canvas first-person gifter
  • the excanvas library to port canvas to Internet Explorer
  • Filament Group’s jQuery plugin using canvas for graphing data tables
  • Raphaël JavaScript Library to make SVG that also works in Internet Explorer
  • SVG Web is a JavaScript library which provides SVG support on Internet Explorer
  • My HTML5 forms demo, including range, date, placeholder, regex validation
  • modernizr
  • HTML5demos.com
  • Video demos
