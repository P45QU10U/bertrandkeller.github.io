Les références du FrameWork CSS Markercss

27/01/2010

Markercss est un FrameWork CSS qui se base sur des références définissant des caractéristiques. Plutôt que de nommer les classes de vos pages HTML avec d'odieuses appellations dont vous ne vous rappeler plus les usages, préférez passer par des classes qui indiquent l'affichage généré.

La méthode de déclinaison de Markercss revient à définir les caractéristiques de vos différentes zones dans les classes. Par exemple si je veux définir une classe en absolu, une largeur de 220px et une hauteur de 370px, j'écris : class=”ab w220 h370”.

Avec cette méthode, il est possible de définir des paddings, des margins, des polices, des couleurs... Ça ressemble à du style en ligne mais en passant pas des classes. Chaque caractéristique est implémentée par une classe spécifique, en poussant le concept, on peut arriver à des quintuples classes voir plus.

Si ce type de fonctionnement correspond bien à certains développeurs ; pour d'autres il est carrément à proscrire. La sémantique disparaît de la page HTML pour être remplacée par du code sans signification. En effet, une classe définit la nature d'un élément, le style appliqué (comme son nom l'indique) indique comment il doit s'afficher en fonction de l'interface. Sur une interface différente avec une feuille de style différente, une zone d'affichage aura d'autres dimensions.

Markercss peut être utiliser dans le cadre d'exercices pratiques pour comprendre le comportement d'une page en fonction de l'appel d'une classe (comment marche un FrameWork) ; cependant, il est nécessaire de passer rapidement sur un modèle de fonctionnement sémantisé afin de coder selon les règles de l'art.
